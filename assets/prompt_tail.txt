# [작업 방식 – 반드시 이 순서로 진행]

❗ 추가 질문 금지

❗ 바로 완성형 초안 제시

---

# 1단계. 자동 분석 + 전략 초안

1. 타겟 모델 (구체적 인물 + 상황)
2. 핵심 문제 (1~2개)
3. 핵심 가치 5가지 (구체적 문장)
4. 구매 트리거 (즉시 구매를 유도하는 요소)
5. 브랜드 톤 (1개)
6. 컬러 방향 (1개)

---

# 2단계. 상세페이지 구조 + 카피 + 이미지 생성 프롬프트 설계

👉 모든 출력은 **image 2 이미지 생성용 프롬프트 중심으로 작성**

각 이미지는 반드시 아래 구조 포함:

---

## ✔ 이미지 내 메시지 구조 (필수)

- 메인 카피 (핵심 메시지 1개)
- 서브 카피 (설명 문장 1~2개)
- 보조 포인트 (3~5개)
- 신뢰/근거 요소 (1개 이상)

👉 한 이미지 = 하나의 설득 완결

---

## ✔ 반복 설계 원칙

동일한 핵심 가치라도 아래 방식으로 반복:

- 상황 변화 (아침 / 저녁 / 직장 / 집 / 혼자 / 가족)
- 표현 변화 (감성 / 기능 / 비교 / 숫자)
- 시각 변화 (모델 / 클로즈업 / 사용 장면 / 결과 장면)

---

# ✔ 섹션별 구성

## 1. Hook (모델 등장)

- 시선 확보
- 핵심 메시지 직관 전달

---

## 2. 문제 공감

- 고객 현실 상황
- 감정 자극
- 반복 공감

---

## 3. 해결 제안 (Before / After)

- 변화 강조
- 선택의 차이 명확화

---

## 4. 핵심 가치 5장 (각각 고밀도)

각 이미지마다:

- 메인 메시지
- 설명 2~3줄
- 보조 포인트 3개
- 신뢰 요소 포함

👉 반드시 다른 방식으로 표현

---

## 5. 신뢰 요소 (강화 필수 구간)

👉 반드시 포함:

- 실제 구매 후기 (상황 기반 문장)
- 구매 평점 (예: ★4.8 / 5.0)
- 누적 판매량 (과장 금지, 사실 기반)
- 인증 / 원산지 / 제조 공정 근거

👉 목적: “의심 제거 → 구매 확신”

---

## 6. 상세 정보 (구매 판단 완결 구간)

👉 반드시 포함 (필수 항목):

- 제품 구성 / 용량
- 원재료 / 주요 성분
- 사용 방법 / 섭취 방법
- 보관 방법
- 제조 방식 / 특징
- 유통기한
- 주의사항

👉 표현 방식:

- 표(Table) 형태 필수
- 아이콘 + 텍스트 병행
- 한눈에 비교 가능하게 구성

👉 목적: “고민 없이 결제”

---

## 7. 구매 전 체크

- 추천 대상
- 비추천 대상
- 섭취/사용 시 주의사항
- 오해 방지 설명

---

## 8. CTA (행동 유도)

- 즉시 행동 유도
- 구매 이유 강조
- 혜택 / 타이밍 강조

# [JSON 최종 스키마 — 반드시 이 형태만 출력]

아래 구조의 **단일 JSON**만 출력하세요. 키 이름·중첩 구조를 유지하고, 값만 맥락에 맞게 채웁니다.

- `meta`: `schema_version` (예: `"1.0"`), `locale` (예: `"ko"`)
- `stage_analysis`: `target_model`, `core_problems` (배열), `core_values` (배열 5개 내외), `purchase_trigger`, `brand_tone`, `color_direction`
- `image_prompts`: 배열, 길이 **10~12**. 각 원소 객체 필드:
  - `index` (1부터 순번)
  - `section_type` (예: `"Hook"`, `"문제공감"`, `"해결제안"`, `"핵심가치"`, `"신뢰"`, `"상세정보"`, `"구매전체크"`, `"CTA"` 등)
  - `width_px` (고정 **860**)
  - `height_px` (권장 범위 내 정수, 이미지 유형별 규칙 반영)
  - `main_copy`, `sub_copy`, `bullet_points` (배열), `trust_or_proof`, `layout_notes`, `typography_notes`
  - `image2_prompt_en` (image 2 생성용 **영문** 핵심 프롬프트 한 덩어리)
- `typography_system`: `headline_font`, `body_font`, `emphasis_font` (문자열)
- `design_finish_rules`: `background`, `layout`, `model_direction`, `ui_elements` (배열)
- `notes` (선택, 짧은 문자열)

**재확인:** 응답 본문은 **JSON 한 덩어리**뿐이어야 합니다. 앞뒤 공백·설명 문 없이 `{` 로 시작 `}` 로 끝나야 합니다.
